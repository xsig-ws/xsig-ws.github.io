    • 1/26(月) 13:20-14:50：
      東京大学名誉教授
      近山　隆
      「情報技術は人類文明をどこに連れて行くのか？」

      20世紀中盤からの情報技術の進展は、学術・産業のみならず、社会・文化にまで 大きなインパクトを与えてきている。そのさらなる進展は、人類文明の根底から の変化をもたらすことになろう。
      好むと好まざるとにかかわらず訪れることになろうこの変化について、どのよう な技術進展が出来し、それが何をもたらすことになるのかを予測してみたい。

  • 招待講演
    • 1/27(火) 13:30-15:00：
      IBM TJ Watson Research Center
      Vijay Saraswat
      Writing Robust Applications in Resilient X10

      In collaboration with Dave Cunningham (Google), Dave Grove, Ben Herta, Arun Iyengar , Olivier Tardieu (IBM TJ Watson), Kiyokuni Kawachiya, Hiroki Murata, Jun Doi, Tatsuhiro Chiba, Mikio Takeuchi, (IBM Japan), Joshua Milthorpe (ANU, IBM) Sara Salem Houda (ANU), Silvia Crafa (U Padua)
      The rise of big data has seen the emergence of various ad hoc parallel, resilient application programming frameworks that support restricted forms of computation e.g. Hadoop/Map Reduce, Pregel/BSP, Millwheel/Streaming. They enable programmers to write application code as fragments of sequential code that are oblivious to any issues of explicit concurrency (race conditions, deadlocks) or distribution (node- or link-failure, data locality) -- these issues are handled by the application framework.
      These frameworks themselves are written in imperative languages like Java and C++ that themselves do not support any notion of distributed, resilient execution. In contrast, we show that it is possible to extend the multi-place, concurrent imperative language X10 in very simple but powerful ways to account for node failure. Resilient X10 permits programs to continue executing even when a place fails. While data and most of the control state at the place is lost, Resilient X10 shows how the global control state of the program can still be recovered in such a way that a key semantic principle, Happens Before Invariance, continues to hold. This principle permits the programmer to write clean code that continues to work in the presence of failure. We show how resilient versions of high performance, parallel application frameworks (such as Main Memory Map Reduce, Global Matrices) can be developed in a simple and elegant fashion in Resilient X10, augmented with resilient storage.

  • チュートリアル1
    • 1/27(火) 11:00-12:30：
      東京大学
      近藤 正章
      エクストリームスケールコンピューティング時代の省電力技術

      エクサスケール時代のHPCシステムでは、消費電力がその設計や実効性能を制約する 最大の要因の一つと考えられている。また、電気料金が運用コストに占める割合も 増大しており、各構成要素の省電力化や、ジョブスケジューリング・計算資源管理 をも含めた電力制御技術の開発が重要な課題となっている。本チュートリアルでは、 プロセッサやメモリ、ネットワークデバイス等の省電力化に関する要素技術動向に ついて概説するとともに、大規模計算機環境におけるシステムレベルでの電力制御 技術についても紹介する。

    • 1/28(水) 11:00-12:30：
      ストレージネットワーキング・インダストリ・アソシエーション日本支部 (SNIA-J) 教育委員会
      親泊 肇
      最新ストレージ技術動向 -ネットワークを中心に、最近ホットなSSD、OSD、Green Storageなどのご紹介-

      ビッグデータ、クラウドコンピューティングという新しいモデルの普及に伴い、 SSDをはじめとした新メディア、Software Defined Network(SDN)と呼ばれる 新しいネットワーク技術の台頭等を背景に、ストレージ技術は一層の進化を 遂げている。本チュートリアルでは、この中からFibre Channel(FC)、iSCSI、 FCoE、InfiniBand(IB)といったストレージネットワーク技術を中心に解説する。 また、ストレージのみならずITの世界でホットなトピックスであるSSD、OSD (Object-based Storage Device)、Green Storageの動向、およびこれらに 対するSNIAおよびSNIA-J(Storage Networking Industry Association Japan Forum)の取り組みについても紹介する。
